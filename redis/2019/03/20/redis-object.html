Reference

1概述

redis基于前文介绍的数据结构构建了一个对象系统，包含如下五种对象:

  1. string
  2. list
  3. hash
  4. set
  5. zset

redis对对象设置多种不同数据结构的实现，以优化在特定场景下的效率

对象结构

// server.h
typedef struct redisObject {
    unsigned type:4;// 类型
    unsigned encoding:4; // 编码
    unsigned lru:LRU_BITS; /* LRU time (relative to global lru_clock) or
                            * LFU data (least significant 8 bits frequency
                            * and most significant 16 bits access time). */
    int refcount; // 引用计数
    void *ptr; // 指向底层实现数据结构的指针
} robj;

1.1 类型与编码实现

  • 查看类型与编码
> type key // 查看类型
> object encoding key // 查看对象编码
  • 编码
// server.h

/* Objects encoding. Some kind of objects like Strings and Hashes can be
 * internally represented in multiple ways. The 'encoding' field of the object
 * is set to one of this fields for this object. */

// 一些对象的底层实现可以有多种方式，例如字符串和哈希
// 如下是类型编码
#define OBJ_ENCODING_RAW 0     /* Raw representation */
#define OBJ_ENCODING_INT 1     /* Encoded as integer */
#define OBJ_ENCODING_HT 2      /* Encoded as hash table */
#define OBJ_ENCODING_ZIPMAP 3  /* Encoded as zipmap */
#define OBJ_ENCODING_LINKEDLIST 4 /* No longer used: old list encoding. */
#define OBJ_ENCODING_ZIPLIST 5 /* Encoded as ziplist */
#define OBJ_ENCODING_INTSET 6  /* Encoded as intset */
#define OBJ_ENCODING_SKIPLIST 7  /* Encoded as skiplist */
#define OBJ_ENCODING_EMBSTR 8  /* Embedded sds string encoding */
#define OBJ_ENCODING_QUICKLIST 9 /* Encoded as linked list of ziplists */
#define OBJ_ENCODING_STREAM 10 /* Encoded as a radix tree of listpacks */

1.2 字符串对象

redis中字符串对象有三种编码方式:

  1. int
  2. embstr
  3. raw

当字符串长度大于一定值时，采用raw编码

// object.c
#define OBJ_ENCODING_EMBSTR_SIZE_LIMIT 44

示例

127.0.0.1:6379> set str 123
OK
127.0.0.1:6379> object encoding str
"int"
127.0.0.1:6379> set str abcdefghijklmnopqrstuvwxyzaaaaaaaaaaaaaaaaaa
OK
127.0.0.1:6379> strlen str
(integer) 44
127.0.0.1:6379> object encoding str
"embstr"
127.0.0.1:6379> set str abcdefghijklmnopqrstuvwxyzaaaaaaaaaaaaaaaaaaa
OK
127.0.0.1:6379> strlen str
(integer) 45
127.0.0.1:6379> object encoding str
"raw"

raw与embstr对比

embstr编码用来保存短字符串,其内存空间连续,分配以及释放内存只需一次操作

  • raw

  • embstr

编码转换

一些情况下对字符串类型数据的修改会导致编码的变化
e.g.

  1. embstr –> raw
  2. int –> raw

对embstr编码的字符串修改会使编码变为raw，redis没有对embstr编码的字符串提供修改操作

1.4 哈希对象

编码方式:ziplist, hashtable
ziplist编码时，相邻的两个entry一个为key，一个为value

编码转换

  • 相关配置 同时满足如下两个条件时使用ziplist编码 hash-max-ziplist-entries // 键值对数量
    hash-max-ziplist-value // 键和值的字符串长度(byte)

1.5 集合对象

编码方式：intset, hashtable
hashtable编码时，使用字典作为底层实现。字典的键为一个字符串对象， 包含了一个集合元素，字典的值全部设置为NULL

编码转换

  • 相关配置 set-max-intset-entries
    所有元素都是整数值，且个数不超过上述配置个数时采用intset编码

1.6 有序集合对象

编码方式：ziplist, skiplist
ziplist编码
相邻的两个entry一个为数据，一个为分值
skiplist编码
此时zset结构中同时包含一个字典和一个跳跃表

  • 跳跃表按分值从小到大保存所有集合元素，每个跳跃表节点保存一个元素
  • 字典保存了一个从成员到分值的映射
    跳跃表和字典通过指针共享相同的元素和分值

编码转换

*相关配置
zset-max-ziplist-entries
zset-max-ziplist-value
相关值同时小于上述配置时采用ziplist编码

1.7 内存回收

redis在其对象系统中采用引用计数实现内存回收

// server.h
truct redisObject {
    // ...
    // 引用计数
    int refcount;
    // ...
} robj;

1.8 对象共享

使用指针指向共享的对象，同时调整共享对象的引用计数值
redis初始化服务器时，创建0-9999共一万个字符串对象
考虑到对象对比的耗时，redis只对包含整数值的字符串对象进行共享
// server.h OBJ_SHARED_INTEGERS常量

1.9 对象空转时长

redisObject结构中lru属性记录量对象最后以此被命令程序访问的时间，可以作为内存回收的依据

> set a 100
> object idletime a
