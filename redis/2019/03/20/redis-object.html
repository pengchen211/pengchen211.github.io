1概述

redis基于前文介绍的数据结构构建了一个对象系统，包含如下五种对象:

  1. string
  2. list
  3. hash
  4. set
  5. zset redis对对象设置多种不同数据结构的实现，以优化在特定场景下的效率

对象结构

// server.h
typedef struct redisObject {
    unsigned type:4;// 类型
    unsigned encoding:4; // 编码
    void *ptr; // 指向底层实现数据结构的指针
    
    // ...    
} robj;

1.1 类型与编码实现

  • 查看类型与编码
> type key // 查看类型
> object encoding key // 查看对象编码
  • 编码
// server.h

/* Objects encoding. Some kind of objects like Strings and Hashes can be
 * internally represented in multiple ways. The 'encoding' field of the object
 * is set to one of this fields for this object. */

// 一些对象的底层实现可以有多种方式，例如字符串和哈希
// 如下是类型编码
#define OBJ_ENCODING_RAW 0     /* Raw representation */
#define OBJ_ENCODING_INT 1     /* Encoded as integer */
#define OBJ_ENCODING_HT 2      /* Encoded as hash table */
#define OBJ_ENCODING_ZIPMAP 3  /* Encoded as zipmap */
#define OBJ_ENCODING_LINKEDLIST 4 /* No longer used: old list encoding. */
#define OBJ_ENCODING_ZIPLIST 5 /* Encoded as ziplist */
#define OBJ_ENCODING_INTSET 6  /* Encoded as intset */
#define OBJ_ENCODING_SKIPLIST 7  /* Encoded as skiplist */
#define OBJ_ENCODING_EMBSTR 8  /* Embedded sds string encoding */
#define OBJ_ENCODING_QUICKLIST 9 /* Encoded as linked list of ziplists */
#define OBJ_ENCODING_STREAM 10 /* Encoded as a radix tree of listpacks */

1.2 字符串对象

redis中字符串对象有三种编码方式:

  1. int
  2. embstr
  3. raw

示例代码

127.0.0.1:6379> set str 123
OK
127.0.0.1:6379> object encoding str
"int"
127.0.0.1:6379> set str abcdefghijklmnopqrstuvwxyzaaaaaaaaaaaaaaaaaa
OK
127.0.0.1:6379> strlen str
(integer) 44
127.0.0.1:6379> object encoding str
"embstr"
127.0.0.1:6379> set str abcdefghijklmnopqrstuvwxyzaaaaaaaaaaaaaaaaaaa
OK
127.0.0.1:6379> strlen str
(integer) 45
127.0.0.1:6379> object encoding str
"raw"
