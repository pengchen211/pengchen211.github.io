    1. 概述

    最近工作中使用了es的snapshot/restore来进行跨集群同步，将步骤整理为此文档。
    说明 ：本文仅适用于es2.3.4与hadoop2.7.0，其他版本请参考es官方文档。

2. 环境准备

2.1. elasticsearch-2.3.4

2.1.1. elasticsearch安装

官网下载地址

为简单起见，示例全部采用默认配置，直接将程序包解压运行

./bin/elasticsearch -d -p pid # 后台运行，将pid输出到pid文件
  • 验证 
    $ curl localhost:9200

    2.1.2. repository-hdfs插件安装

    若采用hdfs作为快照存储介质，需要额外安装插件。2.3.4版本的插件较为特殊，直接按照官方方式进行安装后执行snapshot时会出现依赖相关问题，此处不进行深入分析。直接采用第三方仓库提供的插件。 下载地址

  • 安装 
    $ ./bin/plugin install file:///path/to/elasticsearch-repository-hdfs-2.3.4.BUILD-20160731.045929-24-hadoop2.zip
  • 验证 查看已安装插件列表 
    $ ./bin/plugin list

2.2. hadoop-2.7.0

2.2.1. 下载

官网下载地址

2.2.2. 安装

为了简单起见，示例采用伪分布式安装(pseudo-distributed)，将程序包解压，并修改如下配置文件：

  • etc/hadoop/hadoop-env.sh 
    # 设置java路径
export JAVA_HOME=/path/to/java
  • etc/hadoop/core-site.xml
 <configuration>  
      <property>  
          <name>fs.defaultFS</name>  
          <value>hdfs://localhost:9000</value>  
      </property>  
 </configuration>
  • etc/hadoop/hdfs-site.xml
 <configuration>  
     <property>  
         <name>dfs.replication</name>  
         <value>1</value>  
     </property>  
     <property>  
        <name>dfs.name.dir</name>  
        <value>/tmp/hadoop/2.7.0/name</value>  
    </property>  
    <property>  
        <name>dfs.data.dir</name>  
        <value>/tmp/hadoop/2.7.0/data</value>  
    </property>  
 </configuration>

2.2.3. 配置ssh免密

若ssh loclhost无法登陆本机，则进行如下操作：

$ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa  
$ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys  
$ chmod 0600 ~/.ssh/authorized_keys

2.2.5 启动HDFS

  • 启动
$ ./sbin/start-dfs.sh
  • 验证
    1. 通过jps命令查看相关进程是否存在
    2. 浏览器访问web ui，地址http://localhost:50070 (hadoop2默认为50070端口)

3. 注册仓库

curl -XPUT 'http://localhost:9200/_snapshot/hdfs_repo' -d 
'{
  "type": "hdfs",
  "settings": {
    "uri": "hdfs://localhost:9000",
    "path": "elasticsearch/respositories/my_hdfs_repository",
    "max_restore_bytes_per_sec":"1mb",
    "max_snapshot_bytes_per_sec":"1mb"
  }
}'

注意 ：创建仓库时若出现access_control_exception错误，则说明没有hdfs相关目录的权限，需要先授权。

  • 配置说明

基于HDFS的配置最常用的为示例中几个，uri和path是必不可少，余下两个是限速参数，实际生产环境中也不可或缺。参数详细说明以及其他参数请参考如下文档： Snapshot And Restore HDFS存储配置说明(此为5.2版本文档，2.3同样适用)
